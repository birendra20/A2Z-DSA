position: static (default)

Static is the default. Elements participate in normal flow. Use when you don't need special positioning.

position: relative

Use relative to nudge elements or create a positioning context for absolutely positioned descendants.

Common interview trap: using relative with offsets still preserves the element's original space (it doesn't collapse).

position: absolute

Absolute is ideal for tooltips, dropdowns, or badges that should be positioned over other content.

Parent (position: relative)
Tooltip (position:absolute; top:8px; right:8px)

If no positioned ancestor exists, absolute positions relative to the initial containing block (usually the viewport).

position: fixed

Fixed is used for persistent UI like sticky navbars, back-to-top buttons, or modals that remain in view regardless of scroll.

Scroll the page to see the fixed footer remain pinned to the viewport.

position: sticky

Sticky is great for table headers or sidebars that should remain visible while scrolling within a container. It only works when the ancestor's overflow allows scrolling.

Related Concepts: offsets, containing block, z-index

Offsets (top/right/bottom/left) move positioned elements. The reference for offsets is the nearest positioned ancestor for absolute/fixed/sticky. z-index controls stacking order and creates stacking contexts in certain conditions.

Interview tip: Explain when a new stacking context is created (positioned elements with z-index, opacity less than 1, transform, filter, etc.).

Summary & Interview Tips

  • static: default; normal flow.
  • relative: stays in flow; offsets move visually; useful as positioned ancestor.
  • absolute: removed from flow; positioned relative to nearest positioned ancestor.
  • fixed: removed from flow; positioned relative to viewport.
  • sticky: hybrid; becomes fixed within scroll container after threshold.

Common Interview Mistakes

  • Expecting absolute to be positioned relative to a non-positioned parent (need position:relative).
  • Forgetting that relative keeps original space in flow.
  • Sticky fails when parent overflow is hidden or when no offset specified.
  • Not understanding stacking contexts and z-index behavior.
