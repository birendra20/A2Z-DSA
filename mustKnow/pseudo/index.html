:hover — Visual affordance on pointer hover

What: Styles an element when a pointing device is over it. When to use: hover states for buttons, links, menus. Mistake: relying on hover for critical functionality (mobile doesn't have hover).

:active — Pressed state

What: Applies while the element is being activated (mouse down / key press). When: showing pressed effect. Mistake: expecting :active to persist after click.

:focus & :focus-visible — Keyboard and accessibility focus

What: :focus applies when element receives focus; :focus-visible is shown only when focus should be visible (usually keyboard). When: keyboard navigation, form controls. Mistake: removing focus styles entirely harms accessibility.

Form States — :checked, :disabled, :required, :valid / :invalid

What: Used to style form controls based on state. When: form validation, custom controls. Mistake: styling checked state without keeping semantics for screen readers.

Structural Selectors — :first-child, :last-child, :nth-child(), :not(), :empty, :is()

What: Select elements based on position or relation. When: lists, tables, patterns. Mistake: confusing :nth-child (counts all children) vs :nth-of-type.

:focus-within — Container reacts to child focus

What: Matches an element if any descendant has focus. When: forms, search bars highlighting container. Mistake: expecting purely visual focus without keyboard support.

Pseudo-Elements — ::before, ::after, ::first-letter, ::first-line, ::selection

What: Style or inject content for parts of an element. When: decorative content, affordances, typographic effects. Mistake: using pseudo-elements for content that should be semantic DOM content.

Summary & Interview Tips

This project demonstrates the practical uses and accessibility best practices for pseudo-classes and pseudo-elements.

Quick Notes

  • Pseudo-class: styles based on state (use :).
  • Pseudo-element: styles parts or injects content (use ::).
  • Prefer CSS for visual states and small content insertion; use JS when behaviour or complex DOM is needed.
